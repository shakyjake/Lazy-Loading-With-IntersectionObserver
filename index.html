Lazy Loading with IntersectionObserver

To use, include the lazyloader.js file just before </body> (like this):

<script href="lazyloader.0.4.js" defer="defer"></script>

Then add the "js-image" class to your images, change "src" to "data-src" and add a fallback image inside a noscript element with a class of "nojs-image" (like so):

<img class="js-image" data-src="https://picsum.photos/990/557?image=456" width="990" height="557" alt="Image 456" /> <noscript class="nojs-image"> <img src="https://picsum.photos/990/557?image=456" width="990" height="557" alt="Image 456" /> </noscript>

Width and height attributes are strongly recommended.

This also works for picture elements (change the "srcset" attribute of the source elements to "data-srcset")

Load Effects

You can use CSS transitions with the "js-image", "lazy-loading" and "lazy-loaded" classes to add effects for uninitialised, loading and loaded images respectively.

This demo uses the following:

.js-image { opacity: 0.01; } .lazy-loading { opacity: 0.05; } .lazy-loaded { opacity: 1; transition: all 0.2s linear; }

I haven't tested anything more complex so can make no assurances to its robustness.

Required CSS

You must include the following for the plugin to work:

.lazy-holder, .lazy-responsive { display: block; max-width: 100%; position: relative; } .lazy-responsive > img, .lazy-responsive > picture { bottom: 0; height: 100%; left: 0; position: absolute; right: 0; top: 0; width: 100%; }

Browser Support

The plugin works wherever IntersectionObserver is supported. For browsers that do not support it, there is a non-lazy fallback, provided querySelectorAll is supported (so IE7 and lower won't see any images).

Known Issues

Post-fold images may load early if there's a large pre-fold image as the IntersectionObserver is firing before the pre-fold image has loaded.
